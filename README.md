A space exploration and resource acquisition game. A ship is piloted in 2D space, rotating and thrusting to move. The screen shows a ship, planets, asteroids, black holes, or worm holes generated randomly. The pilot can follow the course generated by the navigation computer in relative safety to collect resources from planets or chart their own course at greater risk and reward!

The game window starts on CLI execution of main.py. Using Pygame a game loop ticks many times per second, handling input, updating objects, and drawing to the screen. Sprites (ship, planets, asteroids, black holes, worm holes) track position/velocity, belong to groups, and interact via collision detection. Timers/frames regulate events; logging snapshots help debug state. I made use of vectors, basic physics, state management, and modular design.

These core ideas map to any interactive system—simulation loops, event handling, real-time state updates, and logging/telemetry. I practiced data structures, OOP design, and debugging with logs—useful in game dev, robotics, UI apps, and backend services that process events continuously.
